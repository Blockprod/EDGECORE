# EDGECORE - Quick Start Guide

Bienvenue dans EDGECORE, un système de trading quantitatif production-grade basé sur l'arbitrage statistique.

## 🚀 Installation Express (3 minutes)

```powershell
# 1. Naviguer au projet
cd c:\Users\averr\EDGECORE

# 2. Créer un environnement virtuel
python -m venv venv
venv\Scripts\activate

# 3. Installer les dépendances
pip install -r requirements.txt

# 4. Vérifier la configuration
python check_health.py

# 5. Copier le template .env
copy .env.example .env

# 6. Éditer .env avec vos credentials (API IBKR, etc.)
# Notepad .env
```

## 📊 Première Session

### A. Découvrir des paires cointegrated

```powershell
python examples_pair_discovery.py
```

Cela va :
- Charger 2 ans de données historiques (AAPL, MSFT, JPM, etc.)
- Tester la cointegration Engle-Granger
- Afficher les paires candidates pour le trading

### B. Lancer un backtest

```powershell
python examples_backtest.py
```

Cela va :
- Charger les données historiques
- Générer des signaux de trading basés sur les Z-scores
- Calculer les métriques de performance (Sharpe, Drawdown, etc.)

### C. Paper Trading (mode sandbox)

```powershell
python main.py --mode paper --symbols AAPL/SPY MSFT/SPY
```

Cela va :
- Connecter à l'broker (en mode sandbox)
- Générer des signaux en temps réel
- Placer des ordres de test
- Journaliser tous les événements

## 📁 Structure Clés

| Dossier | Rôle |
|---------|------|
| `strategies/` | Logique de trading |
| `risk/` | Gestion du risque (INDÉPENDANTE) |
| `execution/` | Abstraction broker (IBKR API, IBKR) |
| `backtests/` | Tests de performance |
| `monitoring/` | Logging structuré |
| `config/` | Configuration YAML |

## ⚙️ Configuration

**Fichiers principaux:**
- `config/dev.yaml` → Environnement développement (sandbox)
- `config/prod.yaml` → Production (live trading)

**Paramètres clés à ajuster:**
```yaml
strategy:
  entry_z_score: 2.0      # Seuil d'entrée (écart-type)
  exit_z_score: 0.0       # Seuil de sortie (mean reversion)

risk:
  max_risk_per_trade: 0.005    # 0.5% du capital par trade
  max_concurrent_positions: 10  # Max positions ouvertes
  max_consecutive_losses: 3     # Kill-switch après 3 pertes
```

## 📝 Workflow Complet

### 1️⃣ Développement
```powershell
python check_health.py              # Vérifier le setup
python examples_pair_discovery.py   # Trouver les paires
python examples_backtest.py         # Backtest historique
```

### 2️⃣ Paper Trading
```powershell
# Éditer config/dev.yaml avec use_sandbox: true
python main.py --mode paper --symbols AAPL/SPY MSFT/SPY
```

Monitor les logs :
```powershell
Get-Content logs/main_*.log -Wait | ConvertFrom-Json | Select-Object timestamp, event, message
```

### 3️⃣ Live Trading (DÉBUTANT SEULEMENT!)
```powershell
# ÉTAPES CRITIQUES:
# 1. Start with 0.1 AAPL maximum
# 2. Monitor risk metrics continuously
# 3. Keep kill-switches active
# 4. Review logs every 30 minutes

python main.py --mode live --symbols AAPL/SPY
```

## 🔒 Sécurité

### ✓ À faire
- Garder `.env` hors du contrôle de version (dans `.gitignore`)
- Charger les credentials depuis les variables d'environnement
- Utiliser le sandbox avant le live
- Tester avec 1% du capital d'abord

### ✗ À ne PAS faire
- Ne pas committer `.env` avec des secrets
- Ne pas tester en live sans backtest d'abord
- Ne pas augmenter la taille de position sans ralentir
- Ne pas ignorer les logs de risque

## 📊 Monitoring

Tous les événements sont loggés en JSON structuré dans `logs/` :

```json
{
  "event_type": "TRADE_ENTRY",
  "timestamp": "2024-01-15T14:30:45Z",
  "symbol_pair": "AAPL_MSFT",
  "z_score": 2.15,
  "position_size": 0.5,
  "entry_price": 42500.00
}
```

Consulter les logs :
```powershell
Get-Content logs/main_*.log | ConvertFrom-Json | ForEach-Object { $_.message }
```

## 📈 Métriques Attendues

Backtest typique (stratégie conservative, 2 ans) :
- **Sharpe Ratio**: 1.5 - 2.0
- **Max Drawdown**: 10% - 15%
- **Win Rate**: 55% - 65%
- **Profit Factor**: 1.3 - 1.8

*Note: Résultats live différeront due au slippage, commissions, et changements de régime.*

## 🆘 Troubleshooting

### ImportError: No module named 'numpy'
```powershell
pip install -r requirements.txt
```

### ConnectionError to IBKR
- Vérifier la clé API dans `.env`
- Vérifier `use_sandbox: true` en dev
- Vérifier la connexion internet

### Pas de paires cointegrated trouvées
- Augmenter le nombre de symboles
- Allonger la période de lookback
- Vérifier que les données sont à jour
