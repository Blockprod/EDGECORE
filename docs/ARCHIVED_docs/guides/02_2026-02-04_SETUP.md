"""
EDGECORE Installation & Setup Guide

This file guides you through the complete setup process.
"""

# ========================================
# STEP 1: ENVIRONMENT SETUP
# ========================================

"""
1. Copy .env template:
   cp .env.example .env

2. Edit .env with your credentials:
   - broker_API_KEY: Your IBKR (or other broker) API key
   - broker_API_SECRET: Your API secret
   - EDGECORE_ENV: Set to 'dev' or 'prod'

3. Create virtual environment:
   python -m venv venv
   source venv/bin/activate  # Unix/Mac
   venv\Scripts\activate     # Windows
"""

# ========================================
# STEP 2: DEPENDENCIES
# ========================================

"""
Install Python 3.11.9 dependencies:
   pip install -r requirements.txt

This includes:
- numpy, pandas, scipy: Data manipulation
- statsmodels: Cointegration testing
- IBKR API: equity broker connectivity
- ib-insync: Interactive Brokers (stub)
- structlog: Structured logging
- pytest: Testing framework
"""

# ========================================
# STEP 3: CONFIGURATION
# ========================================

"""
Configuration files:
- config/dev.yaml: Development settings (sandbox, smaller positions)
- config/prod.yaml: Production settings (live trading, strict risk)

Key parameters:
- strategy.entry_z_score: Entry signal threshold (default: 2.0)
- risk.max_risk_per_trade: Max risk per trade as % of equity (default: 0.5%)
- risk.max_concurrent_positions: Max open positions (default: 10)
- execution.use_sandbox: Use test broker (default: true in dev)
"""

# ========================================
# STEP 4: RUNNING THE SYSTEM
# ========================================

"""
A. BACKTEST (Historical analysis)
   python main.py --mode backtest --symbols AAPL/SPY MSFT/SPY --start-date 2023-01-01

B. PAPER TRADING (Sandbox mode)
   export EDGECORE_ENV=dev
   python main.py --mode paper

C. LIVE TRADING (Real money - EXTREME CAUTION)
   export EDGECORE_ENV=prod
   python main.py --mode live
"""

# ========================================
# STEP 5: MONITORING & LOGGING
# ========================================

"""
All trades and events logged to logs/ directory:
- main_*.log: System events
- strategy_*.log: Signal generation
- execution_*.log: Order submissions/fills

Monitor in real-time:
   tail -f logs/main_*.log | jq .
"""

# ========================================
# STEP 6: TESTING
# ========================================

"""
Run unit tests:
   pytest tests/ -v --cov=.

Tests validate:
- Cointegration detection
- Risk engine constraints
- Position sizing logic
"""

# ========================================
# STEP 7: IMPORTANT REMINDERS
# ========================================

"""
1. NEVER commit .env file with credentials
2. Always test in sandbox/paper mode first
3. Start with minimal position size in live mode
4. Monitor risk metrics continuously
5. Review logs daily for anomalies
6. Test order execution during low volatility
7. Keep kill-switches active (consecutive losses, max drawdown)
"""

# ========================================
# TROUBLESHOOTING
# ========================================

"""
Q: ImportError: No module named 'structlog'
A: Run `pip install -r requirements.txt` again

Q: ConnectionError to broker
A: Check API keys in .env, verify sandbox setting

Q: No cointegrated pairs found
A: Pair trading requires specific regime; test with larger dataset

Q: Backtests are slow
A: Use fewer symbols or shorter time period initially
"""
