## 1. 🏗️ Architecture & Design

### 1.1 Topologie Générale

```
INPUT LAYER (Data)
    ↓ [data/loader.py, data/preprocessing.py]
    ├─ CCXT Exchange (Binance, Kraken)
    ├─ CSV Historical Data
    └─ Caching Layer (Parquet)

ANALYSIS LAYER (Models)
    ↓ [models/cointegration.py, models/spread.py]
    ├─ Engle-Granger Cointegration Testing
    ├─ OLS Spread Modeling
    └─ Half-Life Mean Reversion Estimation

SIGNAL LAYER (Strategy)
    ↓ [strategies/pair_trading.py]
    ├─ Pair Discovery
    ├─ Z-Score Computation
    └─ Signal Generation (Entry/Exit)

CONTROL LAYER (Risk - INDEPENDENT)
    ↓ [risk/engine.py, risk/constraints.py]
    ├─ Position Limits Check
    ├─ Consecutive Loss Tracking
    ├─ Daily Drawdown Kill-Switch
    └─ OVERRIDE Authority (rejects trades)

EXECUTION LAYER (Broker Abstraction)
    ↓ [execution/base.py, execution/ccxt_engine.py]
    ├─ Order Submission
    ├─ Order Cancellation
    ├─ Position Tracking
    └─ Account Balance Fetch

MONITORING LAYER (Logging & Events)
    ↓ [monitoring/logger.py, monitoring/events.py]
    ├─ Structured JSON Logging
    ├─ Trade Event Audit Trail
    └─ Prometheus Metrics Export
```

### 1.2 Patterns & Principles

#### ✅ Strengths

1. **Risk Engine Independence** (Excellente pratique)
   - Risk management est COMPLÈTEMENT découpl du strategy
   - RiskEngine.can_enter_trade() est invoqué AVANT chaque position
   - Impossible pour la stratégie de contourner les contraintes de risque
   - Code d'audit trail immuable pour les rejets

   ```python
   # Exemple: Risk rejects a trade despite signal
   can_enter, reason = engine.can_enter_trade(symbol, size, equity, vol)
   if not can_enter:
       logger.warning(f"Trade rejected: {reason}")
       return  # Signal is ignored
   ```

2. **Modular Architecture**
   - Chaque module (data, models, strategy, risk, execution, monitoring) est isolé
   - Substitution facile (ex: remplacer CCXT par IBKR)
   - Dépendances minimales entre modules
   - Configuration externalisée (YAML)

3. **Structured Configuration**
   - YAML-based (dev.yaml vs prod.yaml)
   - Singleton Settings pattern (global access, thread-safe)
   - Environment-driven (.env)
   - 20+ tunable parameters

4. **Extensive Logging**
   - Structlog JSON output (machine-readable)
   - Events immutables (EventType enum + dataclass)
   - Prometheus-ready export
   - Audit trail for compliance

#### ⚠️ Limitations

1. **Multi-Strategy Support**: Système optimisé pour pair trading uniquement actuellement
2. **Position Sizing**: Basique (pas d'optimisation Kelly, volatility-adjusted stub)
3. **Execution Latency**: Synchrone (pas de latency modeling)
4. **Live Data**: Requires real CCXT connection (pas de résilience)

### 2.2 Models Layer (`models/`)

#### `cointegration.py` - Statistical Testing

**Rôle**: Tester la cointegration Engle-Granger + half-life estimation

**Algorithme: Engle-Granger (2-step)**

1. **Step 1**: OLS regression → residuals

   ```python
   X = [1, x_1, x_2, ..., x_n]
   beta = lstsq(X, y)
   residuals = y - X @ beta
   ```

2. **Step 2**: ADF test (Augmented Dickey-Fuller) sur residuals

   ```python
   adf_result = adfuller(residuals, regression='c')
   is_cointegrated = adf_result[1] < 0.05
   ```

**Code Quality**: ⭐⭐⭐⭐⭐ (Excellent)

- ✅ Handles edge cases (rcond=None for singular matrices)
- ✅ Structlog integration
- ✅ Returns full diagnostic dict
- ✅ Well-commented

**Half-Life Estimation**: AR(1) model

```python
Δspread_t = ρ × spread_{t-1} + ε
half_life = log(2) / |log(ρ)|
```

**Issues**:

- ⚠️ AR(1) assume stationary spread (après OLS residuals)
- ⚠️ Si ρ >= 1, returns None (unit root, not mean-reverting)

#### `spread.py` - Spread Modeling

**Rôle**: Calculer spread = y - (α + β×x), puis Z-score

```python
class SpreadModel:
    def __init__(self, y, x):
        # OLS fit
        X = [1, x]
        beta = lstsq(X, y)
        self.intercept, self.beta = beta
```

```python
def can_enter_trade(self, symbol_pair, position_size, equity, volatility):
    """
    5 checks before allowing trade:
    1. Position limit reached?
    2. Risk % > max_risk_per_trade?
    3. Consecutive losses exceeded?
    4. Daily loss exceeded?
    5. Volatility regime broken?
    
    Returns: (bool, reason)
    """
    checks = [
        (len(self.positions) < max_concurrent, "Max concurrent"),
        (risk_pct <= max_risk, "Risk per trade"),
        (loss_streak < max_losses, "Consecutive loss"),
        (daily_loss % <= limit, "Daily loss"),
        (volatility < threshold, "Volatility regime")
    ]
    
    for check, reason in checks:
        if not check:
            logger.warning(f"Trade rejected: {reason}")
            return False, reason
    
    return True, None
```

**Kill-Switches** (Hard Stops):

- ✅ 3 consecutive losses → HALT all trading

**Strengths**:

### 3.3 Performance Analysis

**Issue**: `find_cointegrated_pairs()` loops over all symbol pairs

```python
for i, sym1 in enumerate(symbols):  # O(n)
```
