# VERSIONING — Runbooks & Prompts critiques EDGECORE

## Objectif
Centraliser le versioning de tous les runbooks et prompts critiques (docs/RUNBOOK.md, tasks/audits/code/, tasks/corrections/).

## Procédure
1. Tout runbook ou prompt critique doit être versionné dans ce dossier (ou référencé ici avec son historique).
2. Utiliser le contrôle de version Git pour chaque modification.
3. Documenter chaque changement dans le changelog ci-dessous.
