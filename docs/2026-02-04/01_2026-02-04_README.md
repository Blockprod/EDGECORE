# EDGECORE

**Production-Grade Quantitative Trading System**

Statistical arbitrage via pair trading (mean reversion). Designed for institutional-quality robustness, capital preservation, and auditability.

## Philosophy

- **Market-Neutral**: Statistical arbitrage eliminates directional risk.
- **Risk-First**: Risk engine is independent and mandatory.
- **Execution-Agnostic**: Pluggable brokers (CCXT for crypto, IBKR for equities).
- **Reproducible**: All decisions logged and auditable.
- **No Magic**: Explicit parameters, no black-box models.

## Architecture

```
strategies/      → Signal generation (pairs, momentum, etc.)
risk/            → Independent risk enforcement
execution/       → Broker abstraction (CCXT, IBKR)
backtests/       → Vectorized performance testing
data/            → OHLCV loading and caching
monitoring/      → Structured logging and metrics
config/          → Environment-based configuration
```

## Quick Start

### Setup

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Configuration

```bash
cp .env.example .env
# Edit .env with your API keys and settings
export EDGECORE_ENV=dev
```

### Run Backtest

```bash
python -c "
from backtests.runner import BacktestRunner
runner = BacktestRunner()
metrics = runner.run(['BTC/USDC', 'ETH/USDC'])
print(metrics.summary())
"
```

### Paper Trading (Sandbox)

```bash
# Set use_sandbox=true in config/dev.yaml
python main.py --mode paper
```

### Go Live (EXTREME CAUTION)

```bash
# 1. Start with minimal position size
# 2. Verify risk engine is active
# 3. Monitor logs continuously
# 4. Verify slippage vs backtest assumptions
export EDGECORE_ENV=prod
python main.py --mode live
```

## Risk Management

**Kill-Switches** (automatic trading halt):

- 3 consecutive losses in a row
- Daily drawdown > 2% (configurable)
- Volatility regime break (historical outlier)
- Max concurrent positions exceeded

**Position Sizing**:

- Max 0.5% risk per trade
- Volatility-adjusted position scaling
- Notional limits per pair

## Monitoring & Logging

Structured JSON logging to `logs/` directory:

- `risk_violations.log` – Risk rejections
- `trades.log` – Entry/exit events
- `execution.log` – Order submissions/fills
- `strategy.log` – Signal generation

All events include:
- Timestamp (ISO 8601)
- Pair/symbol
- Price, quantity, P&L
- Reason/justification

**Prometheus-compatible metrics** (hook in `monitoring/metrics.py`):

- `edgecore_equity`
- `edgecore_max_drawdown`
- `edgecore_sharpe_ratio`
- `edgecore_trades_total`
- `edgecore_risk_violations`

## Performance Expectations

Conservative pair trading strategy, 2-year backtest:

- Sharpe Ratio: 1.5–2.0
- Max Drawdown: 10–15%
- Win Rate: 55–65%
- Monthly Return: 0.5–1.5%

**Actual live performance will differ** due to:
- Slippage (expected ~3–5 bps)
- Commission (exchange + borrowing fees)
- Liquidity (may not fill at limit prices)
- Regime changes (strategy was trained on historical data)
